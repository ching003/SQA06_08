1. Phản biện: Làm sao text có thể thành vector

TRẢ LỜI:

================================================================================
PHẦN 1: QUÁ TRÌNH BIẾN ĐỔI CƠ BẢN - TỪ TEXT ĐẾN VECTOR
================================================================================

Text được chuyển thành vector qua các bước biến đổi toán học:

BƯỚC 1: TEXT → TOKENS (Tokenization)
Ví dụ: "Tuyển lập trình viên Python"
→ Tokenizer tách thành: ["Tuyển", "lập", "trình", "viên", "Python"]
→ Thêm special tokens: ["[CLS]", "Tuyển", "lập", "trình", "viên", "Python", "[SEP]"]

BƯỚC 2: TOKENS → IDs (Vocabulary Mapping)
Mỗi token được ánh xạ sang một số nguyên dựa trên vocabulary đã build sẵn:
"[CLS]"   → 0
"Tuyển"   → 5234
"lập"     → 7821
"trình"   → 3456
"viên"    → 9012
"Python"  → 15678
"[SEP]"   → 1

→ Kết quả: [0, 5234, 7821, 3456, 9012, 15678, 1]

BƯỚC 3: IDs → EMBEDDING VECTORS (Embedding Layer)
Mỗi ID được tra trong Embedding Matrix (30,000 × 768):
- Đây là bảng lookup table được LEARN trong quá trình training
- Mỗi hàng = vector representation của 1 token

ID 0     → [0.23, -0.45, 0.67, ..., 0.12]  (768 chiều)
ID 5234  → [0.89, 0.34, -0.23, ..., 0.56]  (768 chiều)
ID 7821  → [-0.12, 0.78, 0.45, ..., -0.34] (768 chiều)
...

→ Mỗi token bây giờ là một vector 768 chiều

BƯỚC 4: TOKEN VECTORS → CONTEXTUALIZED VECTORS (Transformer Layers)
Các token vectors đi qua 12 layers Transformer:

Input:    [CLS_vec, Token1_vec, Token2_vec, ..., SEP_vec]
   ↓ Layer 1: Self-Attention + Feed Forward
Layer 1:  [CLS_vec_1, Token1_vec_1, Token2_vec_1, ..., SEP_vec_1]
   ↓ Layer 2: Self-Attention + Feed Forward
...
   ↓ Layer 12: Self-Attention + Feed Forward
Output:   [CLS_vec_12, Token1_vec_12, Token2_vec_12, ..., SEP_vec_12]

Self-Attention mechanism:
- Mỗi token "nhìn" tất cả các token khác trong câu
- Cập nhật vector của mình dựa trên context
- Ví dụ: "Python" trong "lập trình Python" vs "rắn Python" → vector khác nhau

BƯỚC 5: CLS TOKEN → SENTENCE VECTOR (Pooling)
Lấy vector của [CLS] token sau layer 12:
CLS_vec_12 = [0.12, -0.34, 0.56, ..., 0.78]  (768 chiều)
     ↓
Dense layer (Linear transformation: 768 → 768)
     ↓
Tanh activation (squash giá trị về [-1, 1])
     ↓
pooler_output = [0.34, -0.12, 0.89, ..., 0.45]  (768 chiều)

→ ĐÂY LÀ SENTENCE EMBEDDING CUỐI CÙNG

TÓM TẮT PIPELINE:
TEXT → (Tokenizer) → TOKENS → (Vocab) → IDs → (Embedding Matrix) → TOKEN VECTORS
     → (12 Transformer Layers) → CONTEXTUALIZED VECTORS → (CLS Pooling) → SENTENCE VECTOR


1.1 Embedding Matrix lấy ở đâu?

TRẢ LỜI NGẮN GỌN (1 CÂU):
> Embedding matrix KHÔNG "lấy ở đâu ra" sẵn; nó là một ma trận tham số được khởi tạo
> ngẫu nhiên và được học dần trong quá trình pre-training của mô hình (ví dụ BERT, PhoBERT).

--------------------------------------------------------------------------------
GIẢI THÍCH CHI TIẾT:

1. EMBEDDING MATRIX LÀ GÌ?
   - Là một ma trận tham số trainable
   - Kích thước: vocab_size × embedding_dim
   - Ví dụ với PhoBERT: 30,000 × 768
   - Mỗi hàng (row) ứng với 1 token ID trong vocabulary

2. KHỞI TẠO BAN ĐẦU (INITIALIZATION)
   Khi bắt đầu pre-training PhoBERT:
   - Embedding matrix được khởi tạo NGẪU NHIÊN
   - Thường theo phân phối: Xavier Uniform / Gaussian / Truncated Normal
   - Công thức khởi tạo Xavier:

     W ~ Uniform(-√(6/(vocab_size + embedding_dim)), √(6/(vocab_size + embedding_dim)))

   - Lúc này: các vector CHƯA có nghĩa gì cả, chỉ là số ngẫu nhiên

   📌 Không có "từ điển vector" nào tồn tại sẵn trước khi train!

3. QUÁ TRÌNH HỌC EMBEDDING MATRIX (Pre-training)

   Trong pre-training PhoBERT (Masked Language Modeling - MLM):

   Ví dụ:
   Câu gốc:  "Tôi học lập trình Python"
   Input:    "Tôi học [MASK] Python"
   Mục tiêu: Dự đoán từ "lập trình"

   Quy trình:
   Step 1: Token IDs → Embedding vectors (lookup từ embedding matrix)
           [5234, 7821, [MASK], 15678]
           ↓
           [vec_5234, vec_7821, vec_MASK, vec_15678]

   Step 2: Các vectors đi qua 12 Transformer layers
           ↓
           [context_vec_5234, context_vec_7821, context_vec_MASK, context_vec_15678]

   Step 3: Dự đoán token bị mask dựa trên context_vec_MASK
           ↓
           Prediction: "lập trình" (token ID: 3456)

   Step 4: Tính loss (Cross-Entropy Loss)
           loss = -log P(token_đúng | context)

   Step 5: BACKPROPAGATION - CẬP NHẬT EMBEDDING MATRIX
           ∂loss/∂W_embedding → cập nhật embedding matrix

           - Nếu dự đoán sai: vector của các token liên quan được điều chỉnh
           - Embedding matrix dần mã hóa quan hệ ngữ nghĩa

4. VÌ SAO EMBEDDING MATRIX "BẮT ĐẦU CÓ NGHĨA"?

   Vì trong quá trình pre-training:
   - Các token xuất hiện trong NGỮ CẢNH GIỐNG NHAU
   - Bị điều chỉnh embedding theo CÁCH GIỐNG NHAU

   Ví dụ:
   Các từ: "Python", "Java", "C++"
   → Thường xuất hiện gần: "lập trình", "code", "developer"
   → Embedding vectors dần GẦN NHAU trong không gian 768 chiều

   Nguyên lý Distributional Semantics:
   > "You shall know a word by the company it keeps."
   > (Bạn hiểu một từ thông qua các từ xuất hiện cùng nó)

5. EMBEDDING MATRIX TRONG SIMCSE CÓ THAY ĐỔI KHÔNG?

   CÓ, nhưng không phải theo cách bạn nghĩ:

   - SimCSE fine-tune TOÀN BỘ model PhoBERT, bao gồm:
     ✓ Embedding matrix
     ✓ 12 Transformer layers (attention weights + feed-forward)
     ✓ Pooler layer (CLS token projection)

   - MỨC ĐỘ THAY ĐỔI khác nhau:
     + Embedding matrix: thay đổi ÍT (đã học tốt trong pre-training)
     + Transformer layers: thay đổi NHIỀU (học cách encode semantic similarity)
     + Pooler layer: thay đổi NHIỀU NHẤT (học cách tóm tắt ý nghĩa câu)

   - Fine-tuning với Contrastive Learning:
     Embedding matrix được tinh chỉnh để phục vụ SEMANTIC SIMILARITY:
     + Câu giống nghĩa → vectors gần nhau
     + Câu khác nghĩa → vectors xa nhau

6. TÓM TẮT - CÂU TRẢ LỜI "ĐINH" ĐỂ PHẢN BIỆN:

   Nếu ai hỏi: "Embedding matrix lấy ở đâu ra?"

   ✅ Trả lời:
   "Embedding matrix là một ma trận tham số được khởi tạo ngẫu nhiên và được học
   dần trong quá trình pre-training bằng backpropagation - giống như mọi weight
   khác trong neural network. Nó KHÔNG phải là dữ liệu có sẵn, mà là KẾT QUẢ HỌC
   từ dữ liệu."

   📌 LƯU Ý QUAN TRỌNG:
   Embedding matrix CHỈ LÀ BƯỚC ĐẦU - vector cuối cùng đại diện cho ý nghĩa câu
   KHÔNG phải trực tiếp từ embedding matrix, mà là CONTEXTUALIZED EMBEDDING sau
   khi đi qua 12 Transformer layers. Đây mới là output mang nghĩa semantic thực sự!


1.2 Transform Layer là gì? Tại sao không chỉ mean embedding vectors?

TRẢ LỜI NGẮN GỌN:
> Transformer sử dụng SELF-ATTENTION để mỗi token "nhìn" toàn bộ tokens khác,
> cập nhật vector dựa trên ngữ cảnh. Mean embedding KHÔNG làm được điều này vì
> mất hết thông tin về vị trí, quan hệ và ngữ cảnh giữa các từ.

--------------------------------------------------------------------------------

1. VẤN ĐỀ CỦA MEAN EMBEDDING

   ❌ Mean embedding = Bag-of-words (mất ngữ cảnh):

   a) "Python" trong "lập trình Python" vs "rắn Python"
      → Cùng 1 vector, không phân biệt được ngữ cảnh!

   b) "Tôi không thích Python" vs "Không có gì tôi thích hơn Python"
      → Cùng tập từ → Cùng vector → Mất nghĩa phủ định/khẳng định!

   c) "Tuyển lập trình viên" vs "viên trình lập Tuyển"
      → Cùng vector → Không phân biệt thứ tự!

2. TRANSFORMER GIẢI QUYẾT BẰNG SELF-ATTENTION

   Công thức: Attention(Q,K,V) = softmax(QK^T/√d_k) × V

   Ví dụ: "Tuyển lập trình viên Python"

   Token "Python" tính attention với tất cả tokens:
   - Chú ý 28% đến "trình"
   - Chú ý 22% đến "lập"
   - Chú ý 18% đến "viên"
   - Ít chú ý đến "Tuyển"

   → Vector mới của "Python" = weighted sum của Value vectors
   → BÂY GIỜ chứa thông tin: "Python trong ngữ cảnh lập trình viên"

3. 12 LAYERS HỌC CÁC MỨC ĐỘ KHÁC NHAU

   - Layer 1-3:  Cú pháp (syntax), cụm từ
   - Layer 4-8:  Ngữ nghĩa (semantics), quan hệ từ
   - Layer 9-12: Ý nghĩa câu (sentence-level), [CLS] tổng hợp toàn bộ

4. TÓM TẮT

   ✅ CÂU TRẢ LỜI PHẢN BIỆN:
   "Mean embedding CHỈ tạo bag-of-words - mất NGỮ CẢNH, VỊ TRÍ, QUAN HỆ từ.
   Transformer dùng Self-Attention để mỗi token 'học' từ TẤT CẢ tokens khác,
   tạo CONTEXTUALIZED EMBEDDINGS - mỗi từ có vector KHÁC NHAU theo ngữ cảnh."

   Ví dụ:
   - Mean: "I love Python" ≈ "I hate Python" ❌
   - Transformer: "I love Python" ≠ "I hate Python" ✅



================================================================================
PHẦN 2: TẠI SAO CÁC CON SỐ NÀY CÓ Ý NGHĨA?
================================================================================

Text được chuyển thành vector thông qua 2 giai đoạn training:

GIAI ĐOẠN 1: PRE-TRAINING (PhoBERT)
- PhoBERT (BERT cho tiếng Việt) được pre-train trên corpus tiếng Việt lớn với 2 task:
  + Masked Language Modeling (MLM): Che ngẫu nhiên 15% tokens, model dự đoán từ bị che
  + Next Sentence Prediction (NSP): Dự đoán 2 câu có liên tiếp nhau không
- Mục tiêu: Học được representation tổng quát về ngôn ngữ tiếng Việt
- Kết quả: Model hiểu được ngữ nghĩa, ngữ pháp, ngữ cảnh của tiếng Việt

GIAI ĐOẠN 2: FINE-TUNING với CONTRASTIVE LEARNING (SimCSE)
Đây là bước QUAN TRỌNG để 2 câu có nghĩa giống nhau tạo ra vector gần nhau:

Phương pháp Supervised SimCSE (sup-SimCSE):
- Input: Bộ dataset gồm các cặp câu (sentence1, sentence2) với label tương đồng
  + Positive pairs: Các câu có nghĩa tương tự (label = 1)
  + Negative pairs: Các câu khác nghĩa (label = 0)

- Quá trình training:
  1. Đưa câu qua PhoBERT encoder → lấy embedding vector (768 chiều)
  2. Tính cosine similarity giữa các cặp vector
  3. Sử dụng loss function (InfoNCE loss) để:
     → Kéo các positive pairs GẦN NHAU (maximize similarity)
     → Đẩy các negative pairs XA NHAU (minimize similarity)
  4. Backpropagation: Cập nhật trọng số của model để học cách tạo vector sao cho:
     - Câu cùng nghĩa → vector gần nhau trong không gian embedding
     - Câu khác nghĩa → vector xa nhau

- Loss function (InfoNCE):
  L = -log(exp(sim(vi, vj+)/τ) / Σk exp(sim(vi, vk)/τ))
  Trong đó:
  + vi, vj+: vector của positive pair
  + vk: vector của negative samples
  + τ: temperature parameter
  + sim: cosine similarity

Ví dụ cụ thể trong quá trình training:
  Câu A: "Tuyển lập trình viên Python"
  Câu B: "Cần developer Python" (positive pair với A)
  Câu C: "Tuyển kế toán viên" (negative pair với A)

  Ban đầu: vector(A) và vector(B) có thể xa nhau
  Sau training: Model học được cập nhật trọng số sao cho:
  - similarity(A, B) → cao (gần 1)
  - similarity(A, C) → thấp (gần 0)

QUY TRÌNH TẠO VECTOR TRONG HỆ THỐNG:
1. Tokenization: Text → tokens bằng PhoBERT tokenizer
2. Encoding: Tokens đi qua 12 layers của PhoBERT đã được fine-tune
   - Input: [CLS] token_1 token_2 ... token_n [SEP]
   - Mỗi token được biến thành vector 768 chiều qua các Transformer layers
   - Đặc biệt: Token [CLS] được fine-tune trong quá trình SimCSE training để chứa thông tin ngữ nghĩa của CẢ CÂU

3. Pooling: Lấy CLS token embedding → sentence embedding 768 chiều
   - ĐÚNG: Lấy pooler_output (CLS token qua dense + tanh)
   - Đây là cách mô hình VoVanPhuc được thiết kế và fine-tune
   - CLS token này đã học cách "tóm tắt" ý nghĩa cả câu qua contrastive learning
4. Normalization: Chuẩn hóa L2 để tính cosine similarity hiệu quả

LƯU Ý VỀ IMPLEMENTATION:
Code hiện tại trong embedding.py (dòng 59-63) ĐANG DÙNG SAI CÁCH:
  outputs = self.model(**inputs)
  embedding = outputs.last_hidden_state.mean(dim=1).squeeze()  # ❌ SAI - dùng mean pooling

CÁCH ĐÚNG theo thiết kế của mô hình VoVanPhuc:
  outputs = self.model(**inputs)
  embedding = outputs.pooler_output  # ✅ ĐÚNG - dùng CLS token

TẠI SAO?
- Mô hình VoVanPhuc được fine-tune với CLS token làm sentence representation
- Trong quá trình contrastive learning, loss function optimize CLS token embedding
- Mean pooling không tận dụng được quá trình fine-tuning này
- Dùng mean pooling = bỏ đi phần "sentence-level" mà SimCSE đã học!

KẾT LUẬN:
Nhờ quá trình fine-tuning với contrastive learning, model VoVanPhuc/sup-SimCSE-VietNamese-phobert-base
đã học được cách biến đổi text thành vector sao cho các câu có nghĩa tương tự (dù khác từ vựng)
sẽ có vector gần nhau trong không gian embedding 768 chiều.
